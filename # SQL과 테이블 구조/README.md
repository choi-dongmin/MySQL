# SQL과 테이블 구조

SQL은 Structured Query Language 의 약자이다.

관계형 데이터베이스가 기본적으로 표의 형태이고 그것이 잘 정리할수 있다는것, 하는것을 구조화 되었다고 하고

영어로는 "Structured" 이다 

"Query" 쿼리라고 하는것은 데이터베이스에게 요청,질의,수정 같은것을 말하는것이다.

"Language" 언어이다.

다시말해 SQL은 표로 작성되는 관계형 데이터베이스를 만들기 위하여 생성, 수정, 변경 등을 하기 위해 

데이터 베이스와의 약속된 언어 라고 생각하면 된다.

## SQL의 특징

1. SQL은 어떤 컴퓨터언어 보다도 쉽다.

2. 주요하다! (SQL은 관계형 데이터를 사용하는 카테고리의 제품들이 공통으로 사용하고 표준화도 잘 되어 있다.)

## 용어

표를 만들기에 앞서 용어 정리를 해보자

```
Table : 표

Row : 행 (보통 행은 데이터의 개수를 나타낸다.)

Column : 열 (보통 열은 데이터의 타입을 나타낸다.)
```


## 키워드

약속된 언어
